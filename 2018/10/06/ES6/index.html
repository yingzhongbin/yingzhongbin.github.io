ES6的语法糖

当解构太复杂时，可以拆分。

《ES 6 精讲4》-方方老师

原有字面量加强

  • 所见即所得，不需要被 new 出来。
    如 [],{},1,2,3,4,5,true,false

  • 更安全的二进制字面量（0b1111101）

  • 更安全的八进制字面量（0o767）
    字符串支持 Unicode

正则表达式字面量添加 Unicode 支持（u 标记）
正则表达式添加 y 标记，支持粘滞匹配

新增数据类型

Symbol 类型 《JS 中的 Symbol 是什么？》

上述代码中，各个种族的值我们并不关心，只要他们独一无二即可。
记住一句话：
Symbol 生成一个全局唯一的值。
迭代

  • 迭代器

  • 生成器（迭代器生成的语法糖）

  • for of迭代器获取的语法糖）
    只有数组，map可以迭代。对象迭代需要设置迭代方法。

对象属性加强

  • 属性定义支持短语法 obj = { x, y }
  • 属性名支持表达式 obj = {[“baz” + quux() ]: 42}
  • 添加 proto 属性，但不建议使用

模块

导入模块,可以随便取名
import defaultExport from “module-name”;

导入模块，将其重命名为name
import * as name from “module-name”;

导入模块，export为原来模块的属性或方法
import { export } from “module-name”;

导入模块，export为原来模块的属性或方法，将其重命名为alias
import { export as alias } from “module-name”;

导入模块，只导入export1 , export2两小块东西
import { export1 , export2 } from “module-name”;

导入模块，只导入export1 , export2两小块东西。export2被重命名为alias2
import { export1 , export2 as alias2 , […] } from “module-name”;

导入默认的和有名字的东西
import defaultExport, { export [ , […] ] } from “module-name”;

导入模块,属性太多，先重命名为name，使用方法为name.xxx
import defaultExport, * as name from “module-name”;

执行module-name中除去方法的代码，不运行方法
import “module-name”;

导出export

导出n个变量
export { name1, name2, …, nameN };

默认导出
export default expression;

将导入的导出去
export * from …;
